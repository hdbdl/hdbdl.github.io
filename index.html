VirtualBox下oracle linux虚拟机安装增强包遇到的问题

发表于 2018-05-21 | 分类于 linux运维

最近需要oracle linux + oracle 11g测试环境。
oracle linux 和virtualbox都是oracle公司的产品，由于之前一直用vmware，安装过程中遇到点问题，做个笔记。
安装完毕oracle linux 系统登录发现鼠标切换很不自由要按快捷键才可以在虚拟机和主机之间切换，对于习惯了vmware的人来说很不习惯，还好virtualbox也提供了增强包。
直接【设备】-【安装增强功能】结果提示要安装gcc更新kernel-devel等操作，按照网上所搜结果依次执行如下几条命令：

安装完毕reboot，执行autorun.sh继续报错提示devel版本不对，原因是更新了系统内核导致
rpm -qa kernel | sort -V | tail -n 1和uname -a
结果不一样，而安装增强包必须一致，则执行一下操作。
ln -s /usr/src/kernels/2.6.32-696.23.1.el6.x86_64 /usr/src/linux
2.6.32-696.23.1.el6.x86_64换成你的linux的内核版本（可直接进入该目录tab键查看）
然后根据autorun.sh提示kernel-uek-devel kernel-uek-devel-4.1.12-61.1.28.el6uek.x86_64
执行yum install -y kernel-uek-devel kernel-uek-devel-4.1.12-61.1.28.el6uek.x86_64
reboot解决问题。
